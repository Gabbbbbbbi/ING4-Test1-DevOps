# Main Purpose of the Lab

The purpose of this lab is to learn how to properly start and structure an IT project by following DevOps and software engineering best practices.
It includes creating a project folder, initializing a Git repository, setting up a Node.js package, writing a basic Node.js script, creating a simple Express web server, documenting the project, and maintaining a changelog.

# Possible Uses in a Company

The concepts covered in this lab are used daily in professional environments:
	•	Initializing new software projects
	•	Structuring repositories according to company standards
	•	Using Node.js to create lightweight web applications or APIs
	•	Documenting projects through README, CHANGELOG, and version control
	•	Ensuring collaboration using Git and GitHub
	•	Using Express to prototype or deploy microservices

# Step in the DevOps Cycle

This lab belongs to the “Code” and “Build” phases of the DevOps lifecycle.
More precisely, it focuses on:
	•	Source code management (Git)
	•	Project initialization and structure
	•	Development of basic application logic (Node.js)
	•	Preparing the project for future CI/CD pipelines
	•	Documentation practices that support collaboration and maintainability
